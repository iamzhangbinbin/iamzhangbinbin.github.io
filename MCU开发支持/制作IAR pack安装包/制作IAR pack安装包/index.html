    制作IAR pack安装包¶

    当使用 IAR 集成开发环境开发 MM32 微控制器时，需安装本支持包，从而实现在 IAR 中选择 MM32 芯片型号、调试以及下载。

    Pack组成¶

    IAR Pack 包含以下内容：

    目录结构如下：

    • debugger 目录包含 ddf、dmac、svd 三种类型文件，用于IAR调试功能。
      • ddf：定义 RAM 地址空间；
      • dmac：辅助 flashloader，当设备的 RAM 很小或者没有内部 RAM 时需要 flash 相关操作；
      • svd：系统视图文件，详见 CMSIS-SVD。
    • device 目录包含 i79、menu 两种类型文件，是设备描述文件，用于在 IAR 选择具体的设备。
      • i79：定义每个具体设备型号文件对各个文件的链接关系
      • menu：芯片的选型菜单
    • flashloader 目录包含 board、flash、mac、out 四种类型文件，用于 IAR 实现下载功能。
      • mac：初始化设备
      • flash：flash 内存空间定义，以及对 out 和 mac 文件的引用
      • out：下载算法文件
      • board：设备的 flash 分配
    • linker 目录包含 icf 文件，用于将用户代码映射到指定地址空间，并生成可执行文件。

    flashloader目录¶

    IAR 下载算法参考 IAR 官方文档，位于 IAR 的安装路径.\arm\doc\FlashLoaderGuide.ENU.pdf。通过 flash loader 将代码从 RAM 不断的搬运到 flash 里面，实现对 flash 的擦除和编程。

    IAR的下载算法由 .board、.flash、.mac、.out 四个文件组成。

    • .flash 文件是一个 XML 文件，向调试器描述相关的闪存属性。
    • .board 文件是一个 XML 文件，使用 IAR 为开发板执行闪存加载所需的信息。
    • .mac 文件是宏文件，实现基于宏的 flash 加载功能。
    • .out 文件是由 IAR 编译器通过对 flash 的配置生成的。

    编译器设置¶

    • 在 IAR->Options->General Options->Target 选择目标 MCU 内核类型，MM32SPIN5630 对应 STAR 内核。
    • 代码下载到 RAM 中执行，需要修改 Linker 配置。在 IAR->Options->Linker->Config 中编辑链接配置文件，如下图所示：
      • Vector Table：修改 .intvec start 的地址为芯片 RAM 的起始地址；
      • Memory Regions：修改 ROM 地址全部为 0，修改 RAM 起始地址加上 0x40，用于中断向量映射，RAM 结束地址和UM一致。
    • 在 IAR->Options->Linker->Output 中配置输出的 out 文件名称、是否包含调试信息等。
    • 下载算法工程没有 main 函数，因此需要修改函数入口，在 IAR->Options->Linker->Library 中 Enter symbol 一栏中输入 “FlashInitEntry”，然后点击保存。
    • 在 IAR->General Options->Output 配置输出文件目录，默认生成 Debug 目录，可根据需要修改。

    生成.out文件¶

    点击 IAR->Project->Make 编译链接当前工程，在输出路径下找到生成的 .out 文件。

    制作 .mac 文件¶

    当设备不支持访问 RAM 或者 RAM 很小时，无法使用常规的闪存加载方式进行下载，通过 .mac 文件实现小部分的闪存加载函数。

    .mac 是 IAR 内部 C-SPY 调试组件配套宏文件，包括预定义系统宏函数，C-SPY 宏语法规范可以参考 arm\doc\EWARM_DebuggingGuide.ENU 文档。

    当前 mac 文件仅保留以下两个 setup 宏函数：

    • execUserFlashInit() ：在调试器将 flashloader 下载到 RAM 前调用一次，且只用于 flash 程序加载，辅助 flashloader 功能函数设置 flash 相关内存映射环境。
    • execUserFlashExit() ：在 flash 下载完成后调用，函数的主要内容是保存状态数据、退出。辅助 flashloader 功能。

    以上两个函数的功能是实现芯片的 flash 的读写、擦除。因此当 flash 的操作没有变化时，就可以直接复用 mac 文件的内容，若对 flash 的操作有变化，那么就需要修改函数的具体实现。

    linker目录¶

    制作 .Icf 文件¶

    ICF 文件定义了堆栈大小、内存空间大小（ ROM 和 RAM 等）、编译器特定的链接指令，将用户代码与具体的内存地址链接并生成可执行文件，用于调试或下载。此文件的制作很简单，可以通过 IAR 工具 生成，只需配置简单的参数。

    • 打开任意一个IAR工程；
    • 选择目标芯片内核，MM32SPIN5630 选择 STAR 内核；
    • 在 IAR->Options->Linker->Config 中编辑链接配置文件，参考 UM 描述，依次编辑中断向量的起始地址，ROM 和 RAM 的起始地址和结束地址，堆和栈大小。如下图所示：

    修改后保存，将其重命名为MM32SPIN5630E.icf。

    debugger目录¶

    制作ddf文件¶

    ddf文件只要主要包含以下三部分内容：

    • #include：微控制器内核架构引用。
    • [Memory]：微控制器内存空间映射。该部分描述总线映射即可，当然也可定义所有内存空间映射，由于所有外设的地址映射会在SVD中定义，建议此处不做定义。
    • [SfrInclude]：包括一个专门用于定义外设寄存器的单独文件。这可以是 DDF 或 SVD 文件，目前引用 SVD 文件。

    制作 .dmac 文件¶

    dmac 文件是基于 C-SPY 宏语言编写的文件。当设备支持 ETM 追踪功能时，需适配 C-SPY 的 I-jet 驱动已支持 ETM 功能，此时需编辑 dmac 文件对设备 flash 的初始化、解锁等功能。

    C-SPY 是用于嵌入式应用的高级语言调试器，它配合 IAR 系统编译器和汇编器使用，完全集成在 IDE 中，提供开发和调试功能。C-SPY 宏语法跟 C 语言类似，支持 C 语言中允许的大多数语句。

    C-SPY 全部语法详见 IAR 的官方手册，位于 IAR 安装路径 .\arm\doc\EWARM_DebuggingGuide.ENU.pdf。

    同 keil pack 包中的 SVD 文件制作方法。

    devices目录¶

    i79 文件和 menu 文件两者是成对存在的，即一个 i79 文件对应一个 menu 文件。

    制作 .i79 文件¶

    i79 文件主要配置微控制器的内核信息、不同系列的产品命名定义，以及对 linker 文件、ddf 文件、flash loader 的引用。

    文件修改需注意如下几点：

    • CHIP name 和 .menu 中的一致。
    • CORE name 是从 IAR IDE 中 Core 的下拉菜单获取的芯片内核名称。
    • 在使用相对路径描述文件时，要使用 \(TOOLKIT_DIR\) 通配符来修饰相对路径。

    制作 .menu 文件¶

    menu 文件基于 xml 格式，描述设备选型以及引用对应的 i79 文件。

    安装验证¶

    上述文件制作完成后，打开IAR的安装目录，将以上文件复制到对应的路径下即可。

    打开任意工程，选择目标芯片信号，测试擦除、下载、调试是否正常。如存在问题，根据提示信息定位到具体文件或代码，分析问题原因并修改，重新安装。
